## proposal draft (1 paragraph each)

### problem
Our team found that the process of finding a suitable restaurant to visit is highly time-consuming and often inaccurate. Currently, users rely on map-based rankings or social media, manually sifting through reviews, photos, and subjective star ratings. Traditional recommendation systems typically rely on collaborative filtering or overall numerical ratings, completely ignoring the rich, nuanced information hidden within textual reviews. Consequently, users frequently spend excessive time only to make suboptimal dining choices. To address this, we aim to build a context-aware recommendation system using the **Google Local Reviews dataset** (from the UCSD recommender systems repository), which contains extensive restaurant metadata alongside a massive corpus of user-written reviews. The core user-facing questions we aim to answer are: *How can we automatically retrieve and rank restaurants that best match a user’s highly specific, natural-language request? How can we proactively recommend places based on their continuous post-visit feedback, or find venues similar to a given favorite?* Furthermore, we want to allow users to specify exactly which aspects or styles (e.g., atmosphere, service quality) they care most about during their initial use, immediately calibrating the system. To make the results immediately actionable, we will visualize these results via an interactive map overlay interface. This is an interesting problem because leveraging detailed text opinions alongside geographical context, explicit initial preferences, and continuous user feedback will enable a significantly more personalized and iterative discovery experience than conventional rating-based platforms alone.

### methods
To construct this system, we will apply core machine learning algorithms from the course syllabus to natural language text data. First, we will use a pre-trained sentence embedding model to convert the unstructured review texts into dense numerical feature vectors. To explore the underlying structure of this data, we will apply **Principal Component Analysis (PCA)** for dimensionality reduction alongside unsupervised learning like **k-means clustering** or **Gaussian Mixture Models (GMM)**. These are appropriate for grouping semantically similar restaurants (e.g., discovering latent categories like "quiet date spots") based on text features. For standard queries, we will embed the search text and use distance metrics like **cosine similarity** to retrieve relevant reviews. To accommodate a user's initial explicit aspect preferences, we will adjust the feature weights of our extracted aspect-level sentiments prior to computing these similarity scores. For finding similar restaurants, we will calculate pairwise distances between aggregated restaurant embeddings. For automated recommendations driven by ongoing post-visit feedback, we will employ **content-based filtering** and **online learning** techniques, dynamically updating a custom "user embedding" profile using the results from their explicit visit ratings to better predict future preferences. Finally, we will train a supervised ranking model—such as **logistic regression**—that combines text similarity, customized aspect weights, historical relevance, and signals like overall star ratings to accurately rank final recommendations, which the user can directly explore on our geographic **map overlay interface**.
